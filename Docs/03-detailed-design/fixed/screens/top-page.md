# 画面詳細設計: トップページ（記事一覧）

## 基本情報

- **画面名**: トップページ（記事一覧）
- **画面ID**: TOP
- **URL**: `/`
- **説明**: 最新記事の一覧を表示するページ
- **対象ユーザー**: すべてのユーザー

## 画面の目的

最新のブログ記事を一覧表示し、ユーザーが興味のある記事を見つけやすくする。

## レイアウト

### 画面構成

```
┌─────────────────────────┐
│       ヘッダー           │
│  - ロゴ                  │
│  - ナビゲーション        │
│  - 検索ボックス          │
├─────────────────────────┤
│                         │
│   記事一覧エリア         │
│  - 記事カード（複数）    │
│  - ページネーション      │
│                         │
│   サイドバー（オプション）│
│  - カテゴリ一覧          │
│  - タグ一覧              │
│                         │
├─────────────────────────┤
│       フッター           │
└─────────────────────────┘
```

### レイアウト説明

- ヘッダー: サイトロゴ、ナビゲーション、検索ボックス
- メインコンテンツ: 記事一覧（グリッドまたはリスト表示）
- サイドバー: カテゴリ一覧、タグ一覧（オプション）
- フッター: コピーライト、リンク等

## 画面要素

### 記事カード

- **要素名**: 記事カード
- **種類**: カードコンポーネント
- **位置**: メインコンテンツエリア
- **説明**: 各記事の情報を表示するカード
- **初期値**: なし
- **必須/任意**: 必須

#### 記事カードの構成要素
- アイキャッチ画像
- タイトル
- 公開日時
- カテゴリ
- タグ（複数）
- 抜粋（オプション）

### ページネーション

- **要素名**: ページネーション
- **種類**: ページネーションコンポーネント
- **位置**: 記事一覧の下部
- **説明**: ページ番号を表示し、ページ遷移を可能にする
- **初期値**: 1ページ目
- **必須/任意**: 必須（記事数が多い場合）

### 検索ボックス

- **要素名**: 検索ボックス
- **種類**: 入力欄
- **位置**: ヘッダー
- **説明**: 記事を検索するための入力欄
- **初期値**: 空文字列
- **必須/任意**: 任意

## バリデーション

### 入力バリデーション

| 項目名 | バリデーション内容 | エラーメッセージ |
|-------|------------------|----------------|
| 検索クエリ | 空文字列でないこと、最大100文字 | "検索キーワードを入力してください" / "検索キーワードが長すぎます" |

## エラーハンドリング

### エラー表示方法

- エラーメッセージを画面に表示
- エラーアイコンを表示（オプション）

### エラーメッセージ

| エラーコード | エラーメッセージ | 表示位置 |
|------------|----------------|---------|
| R001 | ページが見つかりませんでした | 画面中央 |
| A001 | データの取得に失敗しました | 画面中央 |

## 画面遷移

### 遷移元

- なし（トップページ）

### 遷移先

- この画面 → [記事詳細ページ](./article-detail-page.md)（記事カードクリック時）
- この画面 → [カテゴリページ](./category-page.md)（カテゴリクリック時）
- この画面 → [タグページ](./tag-page.md)（タグクリック時）
- この画面 → [検索結果ページ](./search-result-page.md)（検索実行時）

## 処理フロー

1. 画面表示時
   - ビルド時に生成された静的HTMLを配信
   - 記事データを取得（ビルド時に含まれている）
   - 記事一覧を表示

2. ページネーションクリック時
   - 選択されたページの記事を表示
   - URLを更新（オプション）

3. 検索実行時
   - 検索結果ページに遷移

## 状態管理

### 画面状態

- 現在のページ番号
- 表示中の記事一覧

### データ取得

- ビルド時にmicroCMSから全記事を取得
- クライアントサイドでページネーション処理

## レスポンシブデザイン

### デスクトップ（1024px以上）

- 3カラムレイアウト（記事カードを3列表示）
- サイドバーを表示

### タブレット（768px - 1023px）

- 2カラムレイアウト（記事カードを2列表示）
- サイドバーを非表示または折りたたみ

### スマートフォン（767px以下）

- 1カラムレイアウト（記事カードを1列表示）
- サイドバーを非表示

## アクセシビリティ

### キーボード操作

- Tabキーで各要素を順にフォーカス
- Enterキーでリンクを開く
- 矢印キーでページネーションを操作（オプション）

### スクリーンリーダー対応

- 適切な見出し構造（h1, h2等）
- 画像にalt属性を設定
- リンクに適切なaria-labelを設定

## 注意事項

- 記事数が多い場合はページネーションを実装
- パフォーマンスを考慮して、画像の遅延読み込みを実装
