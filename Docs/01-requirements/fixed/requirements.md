# 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
個人ブログサイト

### 1.2 プロジェクトの目的
個人ブログサイトの作成（自分の考えや経験を共有するためのブログ）

### 1.3 プロジェクトの背景
個人の考えや経験をWeb上で発信し、読者と共有するためのブログサイトを構築する

### 1.4 プロジェクトの範囲
- フロントエンドアプリケーションの開発（Next.js）
- microCMSとの連携
- Vercelへのデプロイ
- 基本的なブログ機能の実装
- 注: コンテンツ管理はmicroCMSで行うため、管理画面の開発は不要

## 2. 対象ユーザー

### 2.1 主要ユーザー
一般ユーザー（不特定多数の読者）

### 2.2 ユーザー属性
- 年齢層: 幅広い年齢層を想定
- 技術レベル: 一般的なWeb利用スキル
- 利用環境: PC、スマートフォン、タブレット
- その他の属性: 個人ブログの読者として、様々な背景を持つユーザーを想定

## 3. 機能要件

### 3.1 必須機能
- 記事一覧表示（microCMSから取得）
- 記事詳細表示（microCMSから取得）
- カテゴリ・タグ機能（microCMSで管理）
- 検索機能（フロントエンド実装）
- ページネーション

### 3.2 推奨機能
- 画像表示（microCMSのメディアライブラリから取得）
- マークダウン表示（microCMSのリッチエディタ対応）
- RSS配信
- ソーシャルシェアボタン
- アクセス解析（Google Analytics等）
- サイトマップ生成

### 3.3 将来検討機能
- コメント機能（外部サービス連携）
- いいね・ブックマーク機能
- ニュースレター配信

## 4. 非機能要件

### 4.1 パフォーマンス要件
- レスポンスタイム: ページ読み込み速度3秒以内を目標
- スループット: 一般的な個人ブログのトラフィックに対応
- 同時接続数: 特に制約なし（Vercelの無料プランで対応可能な範囲）
- その他: Next.jsのSSG（Static Site Generation）を活用して高速化

### 4.2 可用性要件
- 稼働率目標: 99%以上
- ダウンタイム許容範囲: メンテナンス時を除き、可能な限り稼働を維持
- メンテナンス計画: 必要に応じて実施（Vercelのメンテナンスに準じる）

### 4.3 拡張性要件
- 将来の拡張性: モジュール設計により、機能追加が容易な構造
- スケーラビリティ要件: Vercelの自動スケーリング機能を活用

### 4.4 互換性要件
- 対応ブラウザ: 最新版の主要ブラウザ（Chrome、Firefox、Safari、Edge）
- 対応OS: Windows、macOS、Linux、iOS、Android
- その他互換性要件: レスポンシブデザインでPC・タブレット・スマートフォンに対応

## 5. 技術要件

### 5.1 使用技術スタック
- CMS: microCMS（ヘッドレスCMS）
- フロントエンド: Next.js（Reactベース、SSG/SSR対応）
- バックエンド: Next.js API Routes（必要に応じて）
- データベース: microCMSが管理（外部データベース不要）
- インフラ: Vercel（Next.jsのデプロイ）
- その他: 
  - microCMS SDK（JavaScript/TypeScript）
  - マークダウン解析ライブラリ（必要に応じて）

## 6. UI/UX要件

### 6.1 デザイン方針
モダン・トレンディなデザインを採用し、読みやすく使いやすいインターフェースを実現する

### 6.2 ユーザビリティ要件
- 直感的なナビゲーション
- 明確な情報階層
- 読みやすいタイポグラフィ
- 適切な余白とレイアウト

### 6.3 アクセシビリティ要件
- 基本的なアクセシビリティ対応（WCAG 2.1 Level AAを目標）
- 適切なコントラスト比
- キーボード操作への対応
- スクリーンリーダーへの配慮

### 6.4 レスポンシブデザイン
- PC・タブレット・スマートフォンに対応
- モバイルファーストの設計思想
- タッチ操作への最適化

## 7. データ要件

### 7.1 データ構造
- 記事データ: microCMSで管理（タイトル、本文、カテゴリ、タグ、公開日時等）
- メディアデータ: microCMSのメディアライブラリで管理
- その他: フロントエンドで管理するデータは最小限に

### 7.2 データ量
- 想定データ量: 個人ブログとして、記事数は数百〜数千件程度を想定
- データ増加率: 月数件〜数十件の記事投稿を想定

### 7.3 データ保持期間
- データ保持期間: 無期限（削除しない限り保持）
- データ削除: microCMSの管理画面から削除可能

### 7.4 データバックアップ
- バックアップ: microCMSのバックアップ機能に依存
- データ復旧: microCMSの機能を利用

## 8. セキュリティ要件

### 8.1 認証・認可
- 管理者認証: microCMSの管理画面で実施（フロントエンド側では不要）
- 読者認証: 不要（公開ブログ）

### 8.2 データ保護
- APIキーの管理: 環境変数で管理し、GitHubにコミットしない
- 個人情報の取り扱い: 個人情報を扱わない設計

### 8.3 通信セキュリティ
- HTTPS通信: Vercelで自動的にHTTPS化
- API通信: microCMS APIはHTTPSで通信

### 8.4 脆弱性対策
- 基本的なXSS対策: Reactのデフォルト機能を活用
- CSRF対策: 必要に応じて実装
- 依存関係の管理: 定期的なアップデートとセキュリティパッチの適用

## 9. 制約事項

### 9.1 スケジュール制約
- 特に制約なし

### 9.2 予算制約
- microCMS: 無料プランまたは有料プラン（要確認）
- Vercel: 無料プランで開始（必要に応じて有料プランへ移行可能）

### 9.3 リソース制約
- 開発リソース: 個人開発を想定
- 運用リソース: 個人運用を想定

### 9.4 その他の制約
- コンテンツ管理はmicroCMSに依存
- フロントエンドはNext.jsで実装
- デプロイはVercelで実施

## 10. 用語集

- **microCMS**: ヘッドレスCMSサービス。コンテンツ管理をAPI経由で提供
- **Next.js**: Reactベースのフレームワーク。SSG（Static Site Generation）とSSR（Server-Side Rendering）に対応
- **SSG**: Static Site Generationの略。ビルド時に静的HTMLを生成する方式
- **Vercel**: Next.jsの開発元が提供するホスティングサービス
- **ヘッドレスCMS**: フロントエンドと分離されたCMS。API経由でコンテンツを提供
- **APIキー**: microCMS APIにアクセスするための認証キー
