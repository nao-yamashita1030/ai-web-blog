# AI駆動開発基盤リポジトリ

このリポジトリは、AI駆動開発を行うための基盤リポジトリです。AIコンサルタントと協力して、Webアプリケーション開発プロジェクトを体系的に進めるためのフレームワークとテンプレートを提供します。

## リポジトリの目的

このリポジトリは、以下の目的で作成されています：

- **基盤リポジトリ**: AI駆動開発のための標準的な構造とプロセスを提供
- **テンプレート集**: 各開発フェーズ用のテンプレートを提供
- **ベストプラクティス**: 開発プロセスを標準化し、品質を向上

## 使い方

### 1. リポジトリのフォーク

新しいプロジェクトを開始する際は、このリポジトリをフォークして使用してください：

1. GitHubでこのリポジトリをフォーク
2. フォークしたリポジトリをローカルにクローン
3. プロジェクト固有の設定を開始

```bash
git clone https://github.com/あなたのユーザー名/ai-dev-project.git
cd ai-dev-project
```

### 2. モードの選択

このリポジトリには2つのモードがあります。チャット開始時にモードを選択してください。

#### 開発モード

**トリガー**: 「開発モードスタート」というプロンプトを送信

**用途**: 実際のシステム開発を進めるモード

**動作**:
- AIコンサルタントとして、各開発フェーズを会話形式で進めます
- 要件定義、基本設計、詳細設計、テスト計画、運用計画、実装、テスト、リリースの各フェーズを段階的に進めます
- `Docs/` ディレクトリ内のドキュメントを作成・更新します
- `.cursorrules` やテンプレートファイルは読み取り専用として扱われます

**使用例**:
```
開発モードスタート
```

#### 計画モード

**トリガー**: 「計画モードスタート」というプロンプトを送信

**用途**: 基盤リポジトリ自体を改善するモード

**動作**:
- 開発用のプロンプトや `.cursorrules` を修正します
- すべてのドキュメント（`.cursorrules`、テンプレート、READMEなど）を修正可能です
- 基盤リポジトリの改善・最適化を行います
- 新しいテンプレートの追加や既存テンプレートの改善
- プロンプトの改善や新機能の追加

**使用例**:
```
計画モードスタート
```

### 3. 開発フェーズ

プロジェクトは以下の8つのフェーズで構成されます：

1. **要件定義** (01-requirements)
2. **基本設計** (02-basic-design)
3. **詳細設計** (03-detailed-design)
4. **テスト計画** (04-test-plan)
5. **運用計画** (05-operation-plan)
6. **実装** (06-implementation)
7. **テスト** (07-testing)
8. **リリース** (08-release)

各フェーズの詳細は `Docs/README.md` を参照してください。

```
.
├── .cursorrules              # AIコンサルタントの設定ファイル
├── .gitignore                # Git除外設定
├── README.md                  # このファイル
├── rules/                     # ルール管理
│   ├── README.md              # ルール一覧
│   ├── commit-message.md      # コミットメッセージのルール
│   └── phase-transition.md    # フェーズ移行時のルール
└── Docs/                      # プロジェクトドキュメント
    ├── 01-requirements/       # 要件定義
    │   ├── draft/             # 検討中
    │   ├── fixed/             # 確定済み
    │   └── template.md        # テンプレート
    ├── 02-basic-design/       # 基本設計
    ├── 03-detailed-design/    # 詳細設計
    ├── 04-test-plan/          # テスト計画
    ├── 05-operation-plan/      # 運用計画
    ├── 06-implementation/     # 実装
    ├── 07-testing/            # テスト
    ├── 08-release/            # リリース
    ├── feedback/               # 基盤リポジトリ改善フィードバック
    │   ├── template.md        # フィードバックテンプレート
    │   └── README.md           # フィードバックの使い方
    ├── status.md              # 作業状況伝言メモ（Git管理外）
    └── README.md              # ドキュメント管理の説明
```

## 作業状況の管理

`Docs/status.md` に現在の作業状況を記録します。新しいチャットを開始すると、AIが自動的にこのファイルを読み込んで、前回の続きから作業を開始します。

`status.md` には以下の情報が含まれます：
- 現在のモード（開発モード / 計画モード）
- 現在のフェーズ
- 作業中の内容
- 次のステップ
- 注意事項・引き継ぎ事項

## モードの切り替え

チャット中にモードを切り替えることができます：

- 開発モードに切り替え: 「開発モードスタート」と送信
- 計画モードに切り替え: 「計画モードスタート」と送信

モードが切り替わると、`Docs/status.md` が自動的に更新されます。

## 開発フロー

1. リポジトリをフォーク
2. ローカルにクローン
3. Cursorでプロジェクトを開く
4. 新しいチャットを開始
5. 「開発モードスタート」と送信して開発を開始
6. AIコンサルタントと協力して各フェーズを進める

## 基盤リポジトリの改善

### 開発モードでのフィードバック記録

開発モードで作業中に、基盤リポジトリ（`.cursorrules`、テンプレート、プロンプトなど）に対する改善点・課題点を見つけた場合：

1. `Docs/feedback/` ディレクトリにフィードバックを記録
2. `template.md` をコピーして新しいファイルを作成
3. 改善点・課題点を記録（問題点、改善提案、優先度など）
4. プロジェクト完了後まで継続して記録

詳細は [Docs/feedback/README.md](Docs/feedback/README.md) を参照してください。

### 計画モードでの改善実施

基盤リポジトリ自体を改善したい場合：

1. このリポジトリ（フォーク元）で作業
2. 「計画モードスタート」と送信
3. `Docs/feedback/` ディレクトリ内のフィードバックを確認
4. フィードバック内容を基に、`.cursorrules` やテンプレートを改善
5. 変更をコミット・プッシュ
