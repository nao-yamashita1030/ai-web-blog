# 画面詳細設計: 検索結果ページ

## 基本情報

- **画面名**: 検索結果ページ
- **画面ID**: SEARCH_RESULT
- **URL**: `/search?q={query}`
- **説明**: 検索結果の一覧を表示するページ
- **対象ユーザー**: すべてのユーザー

## 画面の目的

検索クエリに一致する記事を一覧表示する。

## レイアウト

トップページと同様のレイアウト。検索クエリと結果件数を表示。

## 画面要素

- 検索クエリ（見出し）
- 検索結果件数
- 記事一覧（トップページと同様）

## バリデーション

検索クエリのバリデーション（トップページと同様）

## エラーハンドリング

検索結果が0件の場合、「検索結果が見つかりませんでした」を表示

## 画面遷移

- この画面 → [記事詳細ページ](./article-detail-page.md)

## 処理フロー

1. 検索クエリを取得
2. クライアントサイドで検索実行
3. 検索結果を表示
