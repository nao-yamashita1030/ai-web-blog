# 基盤リポジトリ改善フィードバック

このディレクトリは、開発モードで開発を行っている中で見つけた基盤リポジトリに対する修正点・課題点を記録するためのディレクトリです。

## 目的

開発モードで実際に開発を行っている中で、以下のような改善点を見つけた場合に記録します：

- `.cursorrules` の改善点
- テンプレートの改善点
- プロンプトの改善点
- ディレクトリ構造の改善点
- ドキュメントの改善点
- その他、基盤リポジトリをより良くするための提案

## 使い方

### 開発モードでの記録

1. 開発モードで作業中に改善点を見つけたら、`template.md` をコピーして新しいファイルを作成
2. ファイル名は分かりやすく、日付や内容を含める（例: `improvements-2024-01-15.md`）
3. 改善点を記録
4. プロジェクト完了後まで記録を継続

### 計画モードでの活用

1. プロジェクト完了後、計画モードに切り替え
2. このディレクトリ内のフィードバックファイルを確認
3. フィードバック内容を基に、基盤リポジトリの改善を実施
4. 改善完了後、フィードバックファイルをアーカイブまたは削除

## ファイル命名規則

推奨されるファイル命名規則：

- 日付を含める: `improvements-2024-01-15.md`
- プロジェクト名を含める: `improvements-project-name.md`
- 内容を表す: `template-improvements.md`, `prompt-improvements.md`

## 記録すべき内容

以下のような内容を記録してください：

- **問題点・課題**: 何が問題だったか、何が課題だったか
- **改善提案**: どのように改善すべきか
- **発見したフェーズ**: どのフェーズで発見したか
- **優先度**: 改善の優先度（高/中/低）
- **その他のメモ**: 補足情報や参考情報

## 注意事項

- このディレクトリのファイルは、プロジェクト固有のフィードバックとして管理されます
- フィードバックは定期的に記録することを推奨します
- プロジェクト完了後は、計画モードで改善を実施してください
